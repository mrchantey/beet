# beet-cli

The `beet` command-line interface: a scene controller.

The CLI is itself a beet app. Its only built-in commands are the scene controls (`load`/`clear`/`reset`/`dump`/`call`), and it carries retained state: a loaded scene is rehydrated from a local `.beet/scene.json` on every invocation. The actual commands ship as the loadable `default-cli` scene generated by the `export_scenes` bin, so `beet --help` lists whatever the loaded scene installed.

| Command | Description |
|---------|-------------|
| `run` | Run a no-code BSX site, eg `beet run examples/bsx_site` |
| `run-wasm` | Cargo runner for `wasm32-unknown-unknown` targets |
| `build-wasm` | Build a wasm module and its bindings |
| `export-pdf` | Render a route to PDF |
| `s3-sync` | Sync a directory between the local filesystem and S3 |
| `qrcode` | Generate a QR code (`qrcode` feature) |

```sh
cargo install beet-cli
cargo run -p beet-cli --bin export_scenes
beet load target/scenes/default-cli.json
beet --help
```
